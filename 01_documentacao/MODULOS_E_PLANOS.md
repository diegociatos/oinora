**Funcionalidades**:
- Dados cadastrais (CNPJ, IE, CNAE, endereço)

- Jornadas de trabalho
- Políticas (férias, banco horas, ausências)
- Convenções coletivas (CCT) configuráveis
- Fluxos de aprovação customizáveis

**Limites por plano**: ilimitado em todos os planos.

### 2. USERS · Usuários & Permissões ⭐ todos planos

**Funcionalidades**:
- Convidar usuários por email (magic link)
- 10 papéis (roles) configuráveis
- Audit log completo (filtrado por usuário / recurso)
- Suspensão/reativação
- Transferência de ownership
- Permissões granulares por role

**Limites por plano**:
- **Essencial**: até 5 usuários administrativos
- **Profissional**: até 15 usuários administrativos
- **Premium**: ilimitado

(Empregados como usuários do app são contados separadamente, conforme limite de empregados do plano)

### 3. EMP · Empregados (Ficha de Pessoa) ⭐ todos planos

**Funcionalidades**:
- Ficha completa em 7 abas:
  1. Resumo

  4. Vínculo & cargo
  5. Dependentes
  6. Documentos (upload)
  7. Histórico de movimentações
- Foto + dados básicos visíveis ao próprio empregado
- ASO periódico com lembretes
- Tags customizáveis (★ estrela, pcd, sintaq, etc.)
- Filtros avançados na listagem

**Limites por plano**:
- **Essencial**: 30 empregados
- **Profissional**: 100 empregados

- Acima do limite: cobrança por empregado adicional

## 8. Implementação técnica de planos

### Tabela `tenants.plano` e `tenants.modulos_ativos`

```sql
-- já definida em MODELO_DE_DADOS.md
ALTER TABLE tenants ADD COLUMN plano plano NOT NULL DEFAULT 'essencial';
ALTER TABLE tenants ADD COLUMN modulos_ativos TEXT[] DEFAULT ARRAY['config', 'users', 'emp', 'ats', 'cand', 'mov']::TEXT[];
ALTER TABLE tenants ADD COLUMN limites JSONB DEFAULT '{}'::JSONB;
```

### Helper para checar acesso a módulo

```typescript
// /lib/permissions/modulos.ts
const MODULOS_POR_PLANO = {
  essencial: ['config', 'users', 'emp', 'ats', 'cand', 'mov'],
  profissional: ['config', 'users', 'emp', 'ats', 'cand', 'mov', 'ai_tri', 'onb', 'tre', 'pt', 'fp', 'hc', 'chat', 'es'],
  premium: ['config', 'users', 'emp', 'ats', 'cand', 'mov', 'ai_tri', 'onb', 'tre', 'pt', 'fp', 'ava', 'pesq', 'hc', 'chat', 'es', 'juri'],
};

export async function temAcessoAoModulo(tenantId: string, modulo: string): Promise<boolean> {
  const tenant = await db.tenant.findUnique({
    where: { id: tenantId },
    select: { plano: true, modulos_ativos: true }
  });

  if (!tenant) return false;

  // Verifica add-ons explícitos primeiro
  if (tenant.modulos_ativos.includes(modulo)) return true;

  // Senão, verifica o plano
  return MODULOS_POR_PLANO[tenant.plano].includes(modulo);
}
```

### Middleware Next.js para bloquear rotas por módulo

```typescript
// /middleware.ts
import { NextResponse } from 'next/server';
import { temAcessoAoModulo } from '@/lib/permissions/modulos';

const ROTA_MODULO = {
  '/folha': 'fp',
  '/ponto': 'pt',
  '/juridico': 'juri',
  '/treinamentos': 'tre',
  // ...
};

export async function middleware(req) {
  const session = await getSession(req);
  if (!session) return NextResponse.redirect('/login');

  const modulo = Object.entries(ROTA_MODULO)
    .find(([rota]) => req.nextUrl.pathname.startsWith(rota))?.[1];

  if (modulo && !await temAcessoAoModulo(session.tenant_id, modulo)) {
    return NextResponse.redirect('/upgrade?modulo=' + modulo);
  }
}
```

### Tela de upgrade `/upgrade`

Quando usuário tenta acessar módulo não contratado:
- Banner com "Este recurso faz parte do plano Profissional"
- Comparação dos 3 planos lado a lado
- CTA: "Falar com vendas" ou "Adicionar como add-on"
